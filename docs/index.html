Funzionalità

Tutto il necessario,
niente di superfluo

Progettato per infrastrutture Proxmox multi-server senza clustering, dove la migrazione manuale è rischiosa e lenta.

🔄
Migrazione End-to-End
Stop → Backup → Restore → Start → Cleanup. Il flusso completo è automatizzato e verificato ad ogni singolo passo.
📡
Log in Tempo Reale
WebSocket bidirezionale con barra di avanzamento alimentata dai task reali dell'API Proxmox.
🔑
API Token & Password
Autenticazione via API Token PVEAPIToken o ticket password. Ogni host ha le proprie credenziali salvate in locale.
🖥️
Multi-Host illimitati
Gestisci qualsiasi numero di server Proxmox. Connetti, testa e migra tra qualsiasi coppia di host configurata.
💡
VMID Automatico
Rileva automaticamente il primo ID libero sulla destinazione, evitando conflitti senza aprire l'interfaccia Proxmox.
🧹
Cleanup Controllato
Scegli tu se cancellare sorgente e backup temporaneo. Nessuna azione distruttiva avviene senza conferma esplicita.
Come funziona

Cinque fasi,
zero errori manuali

01
Stop del container sorgente
Il container viene arrestato in modo pulito. Migra LXC attende la conferma dello stato stopped prima di procedere, evitando corruzioni dei dati.
02
Backup sul bridge storage
Viene creato un archivio vzdump compresso (vzdump) sullo storage condiviso (NFS/CIFS), visibile da entrambi gli host. L'avanzamento è mostrato in tempo reale.
03
Restore sull'host destinazione
Il backup viene ripristinato sul nodo di destinazione con il VMID scelto e lo storage selezionato. Il task Proxmox viene monitorato fino al completamento.
04
Avvio opzionale
Se selezionato, il container viene avviato automaticamente sulla destinazione. Altrimenti rimane fermo per permettere verifiche prima del go-live.
05
Cleanup guidato
Dopo la verifica, confermi il cleanup: Migra LXC elimina il container originale e il backup temporaneo dal bridge storage. Tutto sotto il tuo controllo.
Installazione

In produzione
in 3 comandi

Node.js 18+ è l'unico requisito. Gira direttamente sul tuo server linux — container, VM o bare-metal. Nessun account esterno, nessun dato inviato fuori dalla tua rete.

  • Node.js v18+ installato sul server
  • Storage NFS o CIFS montato con lo stesso nome su tutti i nodi Proxmox
  • API Token Proxmox con permessi vzdump, create, delete
  • Porta 3001 raggiungibile (o reverse proxy con NPM)
# Clona e installa
git clone https://github.com/abeggi/migra-LXC /migra
cd /migra && npm install
# Installa ed avvia come servizio systemd
chmod +x manage.sh
sudo ./manage.sh install
sudo ./manage.sh enable
sudo ./manage.sh start
# Apri nel browser
open http://<ip-server>:3001
