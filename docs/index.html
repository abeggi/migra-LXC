Funzionalità

Tutto quello che serve, niente di più.

Pensato per homelab e ambienti Proxmox non in cluster.

Migrazione end-to-end

Stop → Backup → Restore → Start → Cleanup, tutto automatizzato con un click.

API Proxmox native

Autenticazione via password (PVE Ticket) e API Token con privilegi minimi.

Feedback in tempo reale

Avanzamento via WebSocket con barra dinamica basata sui task reali di Proxmox.

Interfaccia moderna

Design glassmorphism reattivo, tema chiaro/scuro, gestione host CRUD dal browser.

VMID intelligente

Suggerimento automatico del primo ID libero sul nodo di destinazione.

Servizio systemd

Script bash pronti per installare e gestire l'app come servizio di sistema.

Installazione

Operativo in 3 minuti.

Node.js v18+ richiesto.

1

Clona il repository

git clone https://github.com/abeggi/migra-LXC /migra

2

Installa le dipendenze

npm install nella cartella del progetto.

3

Installa come servizio

sudo ./manage.sh install && sudo ./manage.sh start

4

Apri il browser

Vai su http://<ip-server>:3001 e aggiungi i tuoi host Proxmox.

Requisiti

Cosa ti serve.

Setup minimo, zero dipendenze esterne.

Node.js v18+ sul server che ospita lo strumento.

Storage condiviso (NFS/CIFS) visibile con lo stesso nome su tutti i nodi Proxmox — es. OVM-2TB.

Permessi Proxmox per vzdump, create, delete e config sui nodi coinvolti.

Nginx Proxy Manager

Per esporre l'app tramite NPM:

1

Forward verso porta 3001 del server.

2

Attiva Websockets Support nella scheda Details — obbligatorio per il log in tempo reale.

3

SSL opzionale — l'app rileva automaticamente ws vs wss.

Sicurezza

Le connessioni ai nodi Proxmox avvengono via HTTPS (self-signed accettati). Le credenziali sono salvate in config/hosts.json — si raccomanda l'uso di API Token con privilegi minimi anziché password utente.
